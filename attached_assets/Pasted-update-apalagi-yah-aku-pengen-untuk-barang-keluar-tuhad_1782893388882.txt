Konsep: Surat Pengeluaran Barang (Reservation List PDF)

Analyzing warehouse document structure (15 seconds)
Ngerti! Itu format SAP PM/MM Reservation List — dokumen resmi yang perusahaan pakai untuk minta barang dari gudang untuk keperluan maintenance. Kita bisa buat versi digital-nya. Izinkan saya jelaskan konsep yang cocok:

Konsep Reservation List PDF (Landscape)
Masalah utama: Ada field yang ada di database kita, ada yang tidak
Field di dokumen	Dari mana?	Status
Reservation No	Auto-generate (TREK-2026-xxx)	✅ Sudah ada
Requested (by/date)	User login + tanggal	✅ Sudah ada
Item Code, Nama, UOM, Location, Qty On Hand, Qty Issued	Database items	✅ Sudah ada
Maintenance Order	Diisi manual saat input	➕ Perlu ditambah ke form
Movement Type	Diisi manual (biasanya 261/281)	➕ Perlu ditambah ke form
Functional Location	Diisi manual (kode lokasi mesin)	➕ Perlu ditambah ke form
Equipment	Diisi manual (kode equipment SAP)	➕ Perlu ditambah ke form
Order Type	Dropdown (PM01/PM02/dll)	➕ Perlu ditambah ke form
Maintenance Activity Type	Dropdown (misal: Corrective/Preventive)	➕ Perlu ditambah ke form
Qty Reserved	= Qty yang diminta	✅ Dari jumlah
Qty Issued	= Qty yang dikeluarkan aktual	✅ Dari jumlah
Valuation Type, Stock Indicator, Serial No	SAP-specific, beda per perusahaan	⬜ Kosong, diisi manual
Department	Dari tujuan/pemohon	✅ Dari field "Tujuan"
Approved By, Requested By, Received By	Tanda tangan manual	⬜ Baris kosong
Rencana implementasi:
1. Tambah field di form Barang Keluar (opsional, bisa dikosongi):

Maintenance Order No : [__________]
Functional Location  : [__________]
Equipment No         : [__________]
Movement Type        : [261 ▼] (dropdown)
Order Type           : [PM01 ▼] (dropdown)

3. Trigger cetak:

Langsung setelah simpan transaksi → tombol "Cetak Reservation List"
Dari halaman Laporan Barang Keluar → ikon print per baris
Yang perlu kamu tentukan:
Karena kita perlu tambah field baru di form Barang Keluar, ada dua pilihan:

Opsi A — Form lebih lengkap → Tambah 6 field SAP ke form input (user isi saat input transaksi)

Opsi B — Field kosong di PDF → Tidak ubah form, field SAP dibiarkan kosong di PDF untuk diisi tangan

Saya rekomendasikan Opsi A — lebih profesional dan datanya tersimpan. Mana yang kamu pilih?
